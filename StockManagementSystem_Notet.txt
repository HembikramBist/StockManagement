Stock Management System 

Project Overview

I have developed a Stock Management System using ASP.NET Core by following Clean Architecture principles.
The main goal of this project is to manage products, stock, sales, and users in a structured, secure, and scalable way.

 1 StockManagement.Core
 This project contains business rules and core logic (NO database, NO UI)
 Application Folder
Contains DTOs and Interfaces used to transfer data and define contracts.

* Interfaces/

  * Used to define service contracts (what a service should do).

* CreateProductDto.cs

  * Used when creating a new product.

* UpdateProductDto.cs

  * Used when updating an existing product.

* ProductResponseDto.cs

  * Used to return product data to UI.

* RegisterModel.cs

  * Used for user registration data.

 Domain Folder
Contains core business entities (database models).
 Entities Folder

* BaseEntity.cs

  * Base class with common fields like Id, CreatedAt.

* Product.cs

  * Represents product table (Name, Quantity, Price).

* User.cs

  * Represents users (Username, Password, Role).

* StockTransaction.cs

  * Represents a sales transaction.

* StockTransactionItem.cs

  * Represents items inside a transaction.

* InventoryTransaction.cs

  * Tracks stock changes.

 Enums Folder

* Stores enum values like roles or status (if used).

2  StockManagement.Infrastructure
This project handles database and external services
 Persistence Folder
Contains database-related code.
 Data Folder

* StockDbConnect.cs

  * Entity Framework DbContext

  * Connects application to SQL Server

  * Contains DbSet<Product>, DbSet<User>, etc.

 Migrations Folder

* Stores Entity Framework migration files

* Used to create/update database tables.

 Service Folder

* ProductService.cs

  * Business logic related to products (optional abstraction layer).
